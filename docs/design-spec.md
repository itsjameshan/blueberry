# 设计规范

> 版本：v1.0 | 创建日期：2026-06-23

---

## 一、设计原则

1. **风格统一**：气象模块沿用现有系统的蓝紫色系浅色主题
2. **渐进增强**：先保证核心功能可用，再逐步丰富细节
3. **响应式**：适配桌面端和移动端
4. **只读优先**：普通用户的气象页面以展示为主，不做复杂交互

## 二、颜色规范

### 2.1 主色调（沿用现有系统）

| 颜色 | 色值 | 用途 |
|------|------|------|
| 主色 | `#6C5CE7` | 按钮、高亮、链接 |
| 次要色 | `#A29BFE` | 边框、次要元素 |
| 背景色 | `#F4F2FF` | 页面背景 |
| 卡片背景 | `#FFFFFF` | 卡片、弹窗 |
| 文字色 | `#636E72` | 正文文字 |

### 2.2 气象模块新增颜色

| 颜色 | 色值 | 用途 |
|------|------|------|
| 预警红色 | `#E74C3C` | 极端气象预警、红色建议等级 |
| 注意黄色 | `#F39C12` | 注意级别建议 |
| 正常绿色 | `#27AE60` | 正常级别建议、成功状态 |
| 信息蓝色 | `#3498DB` | 气象数据展示、信息提示 |
| 温度渐变 | `#E74C3C → #3498DB` | 温度可视化（热→冷） |

## 三、组件规范

### 3.1 卡片

- 圆角：12px
- 阴影：`0 2px 12px rgba(108, 92, 231, 0.08)`
- 内边距：24px
- 背景：白色

### 3.2 按钮

- 默认：白色底 + 蓝紫色文字 `#6C5CE7`，边框 `#A29BFE`
- 悬浮：紫色高亮 `#6C5CE7` 底 + 白色文字
- 圆角：12px
- 高度：40px
- 危险按钮：红色 `#E74C3C`

### 3.3 预警横幅

- 高度：48px
- 背景色根据等级变化（红/黄/绿）
- 文字：白色
- 图标：对应气象图标
- 可关闭

### 3.4 数据卡片（气象数据展示）

- 网格布局，自适应列数
- 每项数据包含：图标 + 数值 + 单位 + 标签
- 数值字体：24px 粗体
- 标签字体：13px 灰色

## 四、布局规范

### 4.1 门户首页

- 居中布局，最大宽度 960px
- 两个入口卡片并排（桌面端）/ 上下排列（移动端）
- 卡片尺寸：320px × 200px
- 卡片包含：图标、标题、简述、进入按钮

### 4.2 气象中心

- 左侧：果园列表（可切换）
- 右侧：选中果园的气象数据
- 顶部：预警横幅（如有）
- 底部：农业建议卡片

### 4.3 果园管理

- 表格布局，与现有管理后台风格一致
- 弹窗表单，与现有用户管理弹窗风格一致

## 五、图标规范

- 使用 Font Awesome（与现有系统一致）
- 气象图标映射：
  - 晴天：`fa-sun`
  - 多云：`fa-cloud-sun`
  - 阴天：`fa-cloud`
  - 雨天：`fa-cloud-rain`
  - 雪天：`fa-snowflake`
  - 大风：`fa-wind`
  - 预警：`fa-exclamation-triangle`
  - 果园：`fa-seedling`
  - 温度：`fa-temperature-half`
  - 湿度：`fa-droplet`

## 六、响应式断点

| 断点 | 宽度 | 布局调整 |
|------|------|---------|
| 桌面 | ≥ 1024px | 完整布局 |
| 平板 | 768px ~ 1023px | 侧边栏折叠 |
| 手机 | < 768px | 单列布局，卡片上下排列 |
