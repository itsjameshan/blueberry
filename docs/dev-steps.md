# 开发步骤规划

> 版本：v1.0 | 创建日期：2026-06-23 | 状态：待执行

---

## 开发原则

- **小步迭代**：每个步骤完成后验证功能正常，再进入下一步
- **向后兼容**：不破坏现有检测系统的任何功能
- **可回滚**：每个步骤的改动可以通过 git 回退

---

## 阶段一：基础设施（Step 1-2）

### Step 1：门户首页

**目标**：新增登录后的统一入口页面

**改动范围**：
- `app.py`：新增 `/portal` 路由，修改登录后跳转目标
- `static/portal.html`：门户首页
- `static/css/portal.css`：门户样式
- `static/js/portal.js`：门户交互

**验收标准**：
- [ ] 登录后跳转到 `/portal`
- [ ] 页面展示两个子系统入口卡片
- [ ] 点击卡片可跳转到对应系统
- [ ] 退出按钮功能正常

---

### Step 2：项目配置与环境变量

**目标**：建立环境变量管理机制

**改动范围**：
- `.env.example`：环境变量模板
- `.env`：实际配置（加入 .gitignore）
- `app.py`：加载环境变量
- `requirements.txt`：新增 `python-dotenv`

**验收标准**：
- [ ] `.env` 文件正确加载
- [ ] API Key 从环境变量读取
- [ ] 现有功能不受影响

---

## 阶段二：果园管理（Step 3-4）

### Step 3：数据库扩展

**目标**：新增气象模块所需的数据库表

**改动范围**：
- `database.py`：新增 gardens、weather_data、weather_alerts 表的创建和操作方法

**验收标准**：
- [ ] 三张新表正确创建
- [ ] CRUD 方法可正常工作
- [ ] 现有表和数据不受影响

---

### Step 4：果园管理后端 + 前端

**目标**：管理员可以创建、编辑、删除果园

**改动范围**：
- `app.py`：新增果园管理 API 路由
- `static/garden.html`：果园管理页面
- `static/css/garden.css`：样式
- `static/js/garden.js`：交互逻辑

**验收标准**：
- [ ] 管理员可创建果园（名称、地点、种植日期）
- [ ] 管理员可编辑/删除果园
- [ ] 管理员可设置生长阶段
- [ ] 普通用户只能查看，不能操作
- [ ] 地点输入后自动调用 GeoAPI 获取 location_id

---

## 阶段三：气象数据对接（Step 5-6）

### Step 5：和风天气 API 封装

**目标**：封装和风天气 API 调用，带缓存机制

**改动范围**：
- `weather/__init__.py`
- `weather/api_client.py`：API 调用封装 + 数据缓存

**验收标准**：
- [ ] 可通过城市名获取 location_id
- [ ] 可获取实时天气数据
- [ ] 数据缓存到数据库，避免重复调用
- [ ] API 调用失败时有合理的错误处理

---

### Step 6：气象数据展示页面

**目标**：展示果园的实时天气、预报、预警等信息

**改动范围**：
- `app.py`：新增气象数据 API 路由
- `static/weather.html`：气象中心首页
- `static/css/weather.css`：样式
- `static/js/weather.js`：交互逻辑

**验收标准**：
- [ ] 选择果园后展示实时天气
- [ ] 展示3天天气预报
- [ ] 展示当前生效的天气预警
- [ ] 展示分钟降水、空气质量、天文信息
- [ ] 所有用户可查看，不可编辑

---

## 阶段四：智能建议与推送（Step 7-8）

### Step 7：农业建议规则引擎

**目标**：根据生长阶段 + 气象数据生成农业建议

**改动范围**：
- `weather/advice_engine.py`：规则引擎
- `app.py`：新增建议 API 路由
- `static/advice.html`：农业建议页面
- `static/css/advice.css`：样式
- `static/js/advice.js`：交互逻辑

**验收标准**：
- [ ] 根据果园生长阶段和当前气象数据生成建议
- [ ] 建议分三个等级（正常/注意/预警）
- [ ] 建议内容具体可操作
- [ ] 页面展示清晰，颜色区分等级

---

### Step 8：邮件预警推送

**目标**：检测到极端气象时自动发送邮件通知

**改动范围**：
- `weather/email_sender.py`：邮件发送
- `weather/scheduler.py`：定时任务
- `requirements.txt`：新增 `APScheduler`
- `.env`：邮件配置

**验收标准**：
- [ ] 定时任务定期拉取各果园天气数据
- [ ] 检测到极端气象预警时发送邮件
- [ ] 邮件内容包含预警信息和农业建议
- [ ] 定时任务异常时有日志记录

---

## 开发顺序总览

```
Step 1  门户首页          ← 入口，改动最小
Step 2  环境变量配置       ← 为后续API调用做准备
Step 3  数据库扩展         ← 为果园管理做准备
Step 4  果园管理           ← 用户可创建果园
Step 5  API封装            ← 对接和风天气
Step 6  气象展示           ← 核心功能上线
Step 7  农业建议           ← 智能分析功能
Step 8  邮件推送           ← 主动通知（可选）
```

每完成一个 Step，更新 `devlog/changelog.md` 记录完成情况。
