# 标注抽检方案

## 1. 抽检目标

根据 TC609-5-2025-04 高质量数据集质量评价规范，对蓝莓成熟度检测数据集进行人工复核，验证标注准确性。

## 2. 数据集概况

| 项目 | 数量 |
| --- | --- |
| 总图片数 | 2731 |
| Train | 2194 |
| Val | 266 |
| Test | 271 |
| 类别数 | 3 (`RipeBlueBerry`, `Semi-RipeBlueBerry`, `UnripeBlueBerry`) |
| 标注框总数 | 40218 |

## 3. 抽检策略

### 3.1 抽样比例

根据 TC609-5-2025-04 标注准确性要求，采用分层随机抽样。本文件记录当前已执行的阶段性抽检结果，不再按 300 张抽检计划表述。

- **计划参考比例**: 10%（约273张），可作为正式申报前的扩展抽检目标。
- **当前已执行抽检量**: 40 张（约占 2731 张的 1.46%）。
- **当前已复核标注框**: 856 个（约占 40218 个目标框的 2.13%）。
- **统计置信度说明**: 当前 40 张抽检可用于候选包阶段的标注质量预评估和问题类型识别，不能等同于 10%/300 张抽检结论；若正式申报要求更高置信度，应补抽至约 273-300 张或按审核方要求重新抽样。

### 3.2 分层规则

| 分层维度 | 分层依据 | 抽样数量 |
| --- | --- | --- |
| 数据分割 | Train/Val/Test | 已复核 train 20 张、val 10 张、test 10 张 |
| 来源 | A_ZENODO / B_KAGGLE | 抽样列表覆盖 A/B 两个公开来源 |
| 类别分布 | Ripe/Semi-Ripe/Unripe | 按图片标注框中出现的三类成熟度覆盖 |

### 3.3 抽样方法

1. 从 Train/Val/Test 按阶段性比例抽取样本。
2. 确保每张图片至少包含一个标注框
3. 优先选择标注框数量较多的图片

## 4. 判定标准

### 4.1 漏标 (Missing Boxes)

- 定义：图片中存在肉眼可见的蓝莓果实但未标注
- 判定：复核人发现果实但标注文件中无对应框
- 严重程度：高

### 4.2 错标 (Wrong Labels)

- 定义：标注框位置正确但类别标签错误
- 判定：标注框包含的果实与标注类别不一致
- 严重程度：中

### 4.3 框偏移 (Box Offset)

- 定义：标注框与实际果实位置偏差过大
- 判定：标注框未完整包围果实或严重偏离
- 严重程度：中

### 4.4 重复框 (Duplicate Boxes)

- 定义：同一果实被多个标注框重复标注
- 判定：两个或以上标注框高度重叠且标注同一果实
- 严重程度：低

### 4.5 类别混淆 (Category Confusion)

- 定义：成熟度类别标注不准确
- 判定：`RipeBlueBerry` / `Semi-RipeBlueBerry` / `UnripeBlueBerry` 之间的误判
- 严重程度：中

## 5. 合格标准

| 指标 | 合格阈值 |
| --- | --- |
| 漏标率 | ≤ 5% |
| 错标率 | ≤ 3% |
| 框偏移率 | ≤ 5% |
| 重复框率 | ≤ 2% |
| 类别混淆率 | ≤ 3% |

## 6. 复核流程

1. **准备阶段**: 生成抽检图片列表，创建复核记录表格
2. **复核阶段**: 复核人逐图检查，记录问题
3. **汇总阶段**: 统计各类问题数量，计算比率
4. **结论阶段**: 判断是否达到合格标准
