# 标注抽检方案

## 1. 抽检目标

根据 TC609-5-2025-04 高质量数据集质量评价规范，对蓝莓成熟度检测数据集进行人工复核，验证标注准确性。

## 2. 数据集概况

| 项目 | 数量 |
| --- | --- |
| 总图片数 | 2731 |
| Train | 2194 |
| Val | 266 |
| Test | 271 |
| 类别数 | 4 (green, semi_ripe, ripe, overripe) |
| 标注框总数 | ~25000+ |

## 3. 抽检策略

### 3.1 抽样比例

根据 TC609-5-2025-04 第5.3.4条标注准确性要求，采用分层随机抽样：

- **基础抽检比例**: 10%（约273张）
- **最低抽检量**: 不少于200张
- **实际抽检量**: 300张（约11%）

### 3.2 分层规则

| 分层维度 | 分层依据 | 抽样数量 |
| --- | --- | --- |
| 数据分割 | Train/Val/Test | 各100张 |
| 来源 | A_ZENODO / B_KAGGLE | A约100张，B约200张 |
| 类别分布 | green/semi_ripe/ripe/overripe | 按类别比例分配 |

### 3.3 抽样方法

1. 从 Train/Val/Test 各随机抽取100张
2. 确保每张图片至少包含一个标注框
3. 优先选择标注框数量较多的图片

## 4. 判定标准

### 4.1 漏标 (Missing Boxes)

- 定义：图片中存在肉眼可见的蓝莓果实但未标注
- 判定：复核人发现果实但标注文件中无对应框
- 严重程度：高

### 4.2 错标 (Wrong Labels)

- 定义：标注框位置正确但类别标签错误
- 判定：标注框包含的果实与标注类别不一致
- 严重程度：中

### 4.3 框偏移 (Box Offset)

- 定义：标注框与实际果实位置偏差过大
- 判定：标注框未完整包围果实或严重偏离
- 严重程度：中

### 4.4 重复框 (Duplicate Boxes)

- 定义：同一果实被多个标注框重复标注
- 判定：两个或以上标注框高度重叠且标注同一果实
- 严重程度：低

### 4.5 类别混淆 (Category Confusion)

- 定义：成熟度类别标注不准确
- 判定：green/semi_ripe/ripe/overripe 之间的误判
- 严重程度：中

## 5. 合格标准

| 指标 | 合格阈值 |
| --- | --- |
| 漏标率 | ≤ 5% |
| 错标率 | ≤ 3% |
| 框偏移率 | ≤ 5% |
| 重复框率 | ≤ 2% |
| 类别混淆率 | ≤ 3% |

## 6. 复核流程

1. **准备阶段**: 生成抽检图片列表，创建复核记录表格
2. **复核阶段**: 复核人逐图检查，记录问题
3. **汇总阶段**: 统计各类问题数量，计算比率
4. **结论阶段**: 判断是否达到合格标准

## 7. 输出交付物

- `evidence/qc/annotation_review_records.csv` - 逐图复核记录
- `evidence/qc/annotation_review_summary.csv` - 复核汇总统计
- `docs/annotation_review_certificate_template.md` - 复核证明
