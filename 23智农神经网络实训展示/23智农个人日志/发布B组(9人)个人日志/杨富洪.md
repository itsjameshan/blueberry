# 蓝莓成熟度检测系统 — 实训记录

## 第一天：图片标注与网页初模版搭建

### 实训内容

#### 1. 数据集准备与图片标注

- 收集蓝莓生长各阶段的图片素材，涵盖成熟（RipeBlueBerry）、半熟（Semi-RipeBlueBerry）、未熟（UnripeBlueBerry）三个类别
- 使用标注工具（Labelme）对蓝莓图片进行目标检测标注，为每个蓝莓果实绘制边界框并标注成熟度类别
- 标注完成后，将数据集按 YOLO 格式导出，划分为训练集和验证集

#### 2. 网页前端初模版搭建

- 使用 HTML5、CSS3 和原生 JavaScript 搭建系统前端页面框架
- 设计并开发登录页面（`login.html`）、首页导航（`index.html`）和基础样式（`static/css/`）
- 初步规划页面结构：单图检测、批量检测、大图检测、管理后台等模块

### 实训成果

- 完成蓝莓图片数据集的标注工作，三类成熟度标签清晰
- 前端页面基础框架搭建完成，页面跳转和基础样式正常

---

## 第二天：链接后端技术栈，确保蓝莓检测功能和天气预警功能正常使用

### 实训内容

#### 1. 后端技术栈搭建

- 使用 Flask 3.1 搭建后端服务，配置 Werkzeug 中间件
- 集成 ONNX Runtime 作为推理引擎，实现蓝莓成熟度检测的核心算法
- 配置 SQLite 数据库，初始化用户表、检测记录表和模型配置表（`database.py`）
- 编写检测推理引擎（`detect_engine.py`），实现图片预处理、ONNX 模型推理、后处理、检测框绘制及大图滑窗裁剪与坐标重建

#### 2. 蓝莓检测功能联调

- 将训练好的 YOLOv11 模型导出为 ONNX 格式（`export_onnx.py`），上传至 `onnx_data/` 目录
- 在管理后台上传并激活 ONNX 模型，确保系统启动时能自动加载
- 联调单图检测、批量检测和大图检测接口
- 验证检测框可视化、成熟度数量统计功能正常

#### 3. 天气预警功能接入

- 调研并接入第三方天气 API
- 在前端页面集成天气预警模块，根据用户地理位置或手动输入显示实时天气和预警信息
- 确保天气数据与蓝莓检测系统页面风格统一，数据刷新正常

### 实训成果

- 后端 Flask 服务运行稳定，数据库连接正常
- 蓝莓检测功能完整可用，单图/批量/大图三种检测模式均测试通过
- 天气预警功能成功接入，页面展示正常
