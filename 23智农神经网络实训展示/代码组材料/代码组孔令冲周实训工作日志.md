- 梳理 3000 images 实训用图片、YOLOv11/YOLOv12 训练结果包和 onnx_data 模型文件之间的关系。

### 帮助同学调试开发环境

- 协助各组同学安装 Trae 和 Trae Solo 开发工具，解决了部分同学在安装过程中遇到的网络连接和权限配置问题。
- 安装并配置 LabelME 标注工具，确保所有同学能够正常启动和使用。
- 作为代码组，派遣组员帮助各组解决技术问题，及时反馈各组遇到的问题。

### 标注规范制定与培训

- 向各组成员讲解如何使用 LabelME 进行蓝莓成熟度标注，包括：
  - 标注框的绘制方法与快捷键操作
  - 三类成熟度标签（RipeBlueBerry、Semi-RipeBlueBerry、UnripeBlueBerry）的判别标准
  - 标注质量要求（框体贴合目标、无明显偏移）
- 建立统一的标注数据提交目录结构，确保后续数据收集流程顺畅。
- 制定图片命名规范，统一格式为 `[拍摄日期]_[编号].jpg`，避免命名冲突。
- 设计并维护标注数据收集表，记录各组提交的标注进度和数据质量。

### 蓝莓成熟度标注

- 完成代码组A本组 50-60 张蓝莓图片的成熟度标注，覆盖三类成熟度目标。
- 接收各组提交的标注数据，进行初步质量检查，退回不合格的标注并反馈修改意见。

### 数据格式转换与数据集划分

- 编写脚本将各组提交的 LabelME 标注格式（JSON）统一转换为教师指定的新 YOLO 训练格式（.txt），转换内容包括：
  - 坐标归一化（从像素坐标转为 0-1 相对坐标）
  - 类别 ID 映射（RipeBlueBerry → 0, Semi-RipeBlueBerry → 1, UnripeBlueBerry → 2）
- 完成 train/val/test 数据集划分，确保各类别样本分布均衡。
- 生成 `data.yaml` 和类别映射表，供后续训练使用。

### 夜间训练准备

- 协助整理各组标注数据，准备 YOLO 训练命令、模型导出和 ONNX 兜底方案。
- 编写夜间训练任务单，记录训练启动参数和预期结果。

> 以下为代码组在完成教学计划要求的基础上，自主扩展的天气系统模块开发内容。

#### 项目文档体系搭建

- 创建项目文档目录 `docs/`，编写需求文档、技术规格文档、设计规范文档和开发步骤规划。

- 编写 `CLAUDE.md` 项目指引文件，方便后续开发者快速了解项目结构。

#### 门户首页开发（Step 1）

- 开发 `static/portal.html` 门户首页，展示蓝莓检测系统和气象中心两个子系统入口卡片。
- 添加 `/portal` 路由，显示当前用户信息和退出按钮。

#### 环境变量配置（Step 2）

- 创建 `.env` 和 `.env.example` 文件，配置和风天气 API Key 和 SMTP 邮件服务信息。
- 使用 `python-dotenv` 加载环境变量，确保敏感配置不提交到代码仓库。

#### 数据库扩展（Step 3）

- 在 `database.py` 中新增三张数据表：
  - `gardens` — 果园信息表（名称、地址、经纬度、生长阶段）
  - `weather_data` — 天气数据缓存表
  - `weather_alerts` — 天气预警记录表
- 实现果园 CRUD 函数和天气数据相关函数。

#### 果园管理后端与前端（Step 4）

- 在 `app.py` 中新增果园管理 API（`/api/gardens`），支持 GET/POST/PUT/DELETE 操作。
- 新增天气数据 API（`/api/weather/<garden_id>`）。
- 开发 `static/garden.html` 果园管理页面，包含卡片列表和弹窗表单。
- 开发 `static/weather.html` 气象中心页面，展示果园列表和气象数据。

#### 和风天气 API 封装（Step 5）

- 创建 `weather/` 模块目录，封装和风天气 API：
  - `lookup_city()` — 城市查询
  - `get_realtime_weather()` — 实时天气
  - `get_forecast_3d()` — 3天预报
  - `get_weather_warnings()` — 天气预警
  - `get_minutely_precipitation()` — 分钟降水
  - `get_air_quality()` — 空气质量
  - `get_full_weather()` — 完整天气数据
- 解决 API 连通性问题，配置新的 API Host，增加错误处理使免费版不支持的接口优雅降级。

#### 气象数据展示页面（Step 6）

- 更新 `/api/weather/<garden_id>` API，添加缓存机制（1小时内使用缓存数据）。
- 前端展示实时天气数据（温度、体感温度、湿度、风速）和 3 天预报。
- 实现自动查询并缓存 location_id。

#### 农业建议规则引擎（Step 7）

- 创建 `weather/advice_engine.py`，根据蓝莓四大生长阶段（休眠期、萌芽期、花期、果期）和气象条件生成针对性农业建议。
- 集成到气象数据 API，前端展示建议卡片，区分 warning 和 info 两个级别。
- 后续扩充规则：倒春寒预警、高温灼果预警、连续降水检测、温度剧变预警等。

#### 邮件预警推送（Step 8）

- 创建 `weather/email_sender.py`，使用 SMTP 发送气象预警邮件。
- 当检测到气象预警时，自动向果园管理者发送邮件通知。
- 优化 HTML 邮件模板，采用品牌色和预警色设计，提升邮件可读性。

#### 预警统计与自定义阈值

- 新增预警统计 API（`/api/alerts/stats`），按级别、果园、日期统计。
- 历史预警页面添加 Chart.js 统计图表（环形图、柱状图、折线图）。
- 新增用户自定义预警阈值功能，支持设置高温、低温、高湿度、低湿度、大风、强降雨阈值。

#### 邮箱功能补全

- 给 `users` 表添加 `email` 字段，实现邮箱设置 API 和前端弹窗 UI。
- 验证邮件预警全链路（users.email → email_sender → SMTP）正常。

本次实训不仅提升了我的前后端开发能力，也让我深入了解了智慧农业系统的实际应用场景，对 API 集成、规则引擎设计、邮件推送、智能体辅助开发等工程实践有了更深刻的理解。同时，作为代码组A成员，在帮助各组解决技术问题的过程中，也锻炼了团队协作和沟通能力。
